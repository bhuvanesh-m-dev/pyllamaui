PyLlamaUI Documentation

PyLlamaUI is a lightweight, privacy-focused desktop GUI designed to run and interact with Large Language Models (LLMs) locally. It serves as a bridge between powerful local models and an accessible interface, eliminating the need to rely on the command line or cloud-based services.

Core Philosophy

The core philosophy is pure on-device AI—ensuring no data collection, no telemetry, and zero internet dependency after the initial setup. Your data never leaves your machine.

Privacy is not a feature; it is the foundation. PyLlamaUI operates entirely offline, ensuring your conversations and data remain strictly local.

Features & Capabilities

The application is engineered for a seamless developer experience with several robust features designed to maximize productivity while maintaining complete privacy.

Interactive Chat UI

A clean, responsive interface available as a standalone desktop app or directly inside VS Code. The interface docks neatly alongside your code, allowing you to ask questions, generate snippets, and interact with your local models without leaving your development environment.

Offline Markdown Rendering

Real-time formatting for rich text elements including:

  • Bold and italic text styling
  • Headers and hierarchical structure
  • Ordered and unordered lists
  • Blockquotes and callouts
  • Tables and structured data

All rendering happens locally without internet connectivity.

Code Capabilities

Advanced code handling with syntax highlighting for multiple languages:

  • Python
  • JavaScript
  • JSON
  • Bash/Shell

Features include one-click copy buttons that preserve proper indentation and formatting when transferring code to your active files.

Model Flexibility

Native support for inference with any Ollama-compatible model, including but not limited to:

  • TinyLLaMA
  • Google Gemma
  • DeepSeek-Coder
  • Meta Llama 3
  • Mistral

Architecture & Tech Stack

PyLlamaUI is built on a modern, open-source stack designed for low resource overhead and maximum compatibility.

Overview

The architecture follows a modular design separating concerns between backend logic, frontend presentation, and AI inference engines.

Backend

Python 3.10+ handles the core logic and REST API communications. The backend manages model interactions, state management, and serves as the intermediary between the user interface and the Ollama inference engine.

Frontend / GUI

Two deployment options for the user interface:

  • Desktop Application: Built using the customtkinter framework for native cross-platform desktop experiences
  • VS Code Extension: Utilizes the VS Code Webview API for seamless IDE integration

AI Engine

Powered entirely by Ollama, which manages the local model pulling and inference execution. Ollama provides a unified interface to various open-source large language models.

Installation Guide

Setting up the environment requires a few prerequisites. Ensure your system meets the requirements before proceeding with installation.

Prerequisites

  • Python 3.10 or higher
  • Running instance of Ollama
  • Sufficient disk space for model storage

Python Package Installation

The most direct route for developers is via pip: 

pip install pyllamaui

For system-wide installation, consider using: 

pip install --user pyllamaui

VS Code Extension

Available directly from the Visual Studio Marketplace for an integrated coding assistant experience. Search for PyLlamaUI in the Extensions panel or install via the command line: 

code --install-extension bhuvanesh-m-dev.pyllamaui

Standalone Binaries

Downloadable executables are available via the GitHub releases page:

  • Windows: .exe installer (Currently in testing phase)
  • Linux: .deb packages for Debian/Ubuntu-based systems, including custom distributions like ZentoraOS (Currently in testing phase)

Usage Guide

Operating the interface requires a simple three-step workflow designed for rapid deployment and minimal configuration.

Quick Start

  1. Start the Engine: Ensure the Ollama background service is running on your machine. 
    ollama serve
  2. Pull a Model: Use the terminal to download your preferred model. 
    ollama run llama3
  3. Launch & Chat: Open PyLlamaUI, select your downloaded model from the integrated dropdown menu, and begin your secure offline conversation. 
    pyllamaui --launch

Workflow Integration

In VS Code extension mode, the chat panel docks alongside your code editor. This allows you to:

  • Ask questions about your codebase
  • Generate code snippets contextually
  • Copy formatted code blocks directly into active files
  • Maintain conversation history across sessions

Settings & Configuration

The configuration within PyLlamaUI is designed to be minimal and plug-and-play. No complex configuration files are required for basic operation.

Available Options

  • Model Selection: Dynamically switch between active Ollama models via the integrated dropdown selector
  • Theme Toggle: Switch between light and dark interface modes for user comfort
  • Chat History: Manage standard chat history parameters including conversation persistence and context window size

Roadmap & Future Work

The development pipeline is focused on expanding local intelligence capabilities while maintaining the core philosophy of privacy and offline operation.

Upcoming Milestones

Agentic Mode — Introducing autonomous, reasoning-driven workflows natively in PyLlamaUI and its CLI counterpart. This feature will enable self-directed task execution and complex multi-step reasoning without user intervention.

Stabilization — Moving the Windows .exe and Linux .deb packages from their current testing phases into stable, official releases. This includes enhanced installer validation and system integration.

UX Enhancements — Continuous improvements to interface smoothness, output formatting, and accessibility features.

Contributing Guide

As a 100% free and open-source tool led by Bhuvanesh M, with testing and quality assurance supported by Mohamed Mohideen Saajid S and Mohan Kumar S, community involvement is highly encouraged.

Contribution Workflow

  1. Fork the main repository on GitHub: 
    https://github.com/bhuvanesh-m-dev/pyllamaui
  2. Clone your fork locally and create a new feature branch: 
    git clone https://github.com/your-username/pyllamaui.git
git checkout -b feature/amazing-feature
  3. Commit your updates with descriptive messages: 
    git commit -m "Add: amazing feature description"
  4. Push to your origin branch and submit a Pull Request to the main project: 
    git push origin feature/amazing-feature

Code Standards

All contributions should maintain the existing code style, include appropriate documentation, and ensure compatibility with Python 3.10+ and the customtkinter framework.
